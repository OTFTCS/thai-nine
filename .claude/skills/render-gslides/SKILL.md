# /render-gslides

Generate watercolour sketch illustrations, render the PPTX lesson deck, upload to Google Slides, and apply the Thai font pass. Produces a shareable Google Slides presentation with contextual AI-generated images.

## Arguments
The user provides a lesson ID: `M01-L002`

Parse the argument from the user's message. The lesson ID format is `M\d{2}-L\d{3}`.

## Prerequisites

Before running this skill, the lesson must have:
- `{ID}-script-master.json` — the canonical lesson script
- `{ID}-deck-source.json` — slide definitions (generated by `render_lesson_deck.py`)

If `deck-source.json` doesn't exist yet, Step 1 will create it. If it already exists with `imageUsage: "generated-ai"` entries, those will be preserved.

## Full workflow

### Step 1: Read lesson content and select slides for images

1. Set lesson directory: `course/modules/{module}/{lesson}/` (e.g., `course/modules/M01/L002/`).
2. Read `{ID}-deck-source.json` if it exists. If not, run the renderer first:
```bash
python3 course/tools/render_lesson_deck.py --repo-root . --lesson {ID}
```
3. Read the generated `deck-source.json`. Review every slide's `id`, `role`, `title`, and `visualStrategy`.
4. Read `{ID}-script-master.json` for lesson context (title, objective, section summaries).

### Step 2: Write image prompts

For each slide, decide whether it should get an AI-generated image. Guidelines:

**Always include (unless content is too abstract):**
- Opener (slide 1) — scene-setting illustration for the lesson theme
- Roleplay — the scenario setting (cafe, hotel, market, etc.)
- Teaching slides where the content is concrete and visual (countries, places, food, people, actions)

**Usually include:**
- Recap — a warm "looking back" journal-style image
- Teaching slides with contextual themes (introductions, shopping, travel)

**Usually skip:**
- Objectives — pure text list
- Closing — minimal text slide
- Pronunciation — abstract phonetics (mouth positions, tone marks)
- Teaching slides about very abstract grammar (particles, sentence structure)

**No cap on images** — be generous. 6-10 images per lesson is normal.

For each selected slide, edit `deck-source.json` directly:
1. Set `visualStrategy.imageUsage` to `"generated-ai"`
2. Add `visualStrategy.imagePrompt` with a vivid, specific scene description

**Image prompt rules:**
- Write in English only (no Thai characters)
- Describe a specific scene, not an abstract concept
- Include setting details (Bangkok street, market stall, cafe, hostel)
- Include people doing things (greeting, pointing, exchanging, chatting)
- Include atmosphere (warm light, sunset, lanterns, tropical trees)
- The style prefix is added automatically — don't include "watercolour" in the prompt
- Add "no text or writing in the image" only if the scene might produce signage
- Keep prompts 1-2 sentences, under 200 characters

**Good prompts:**
- "A young Thai man and a Western traveller shaking hands in a sunlit Bangkok park, tropical trees in the background"
- "Three small watercolour vignettes side by side: a golden Thai temple spire, the Tower Bridge of London, and the Statue of Liberty, like stamps in a passport"
- "A hand-drawn travel journal lying open on a wooden desk, with small sketches of a wai greeting and a Thai flag, warm afternoon light"

**Bad prompts (too generic or abstract):**
- "Thai language learning" (no scene)
- "Large triplet card for ชื่อ" (this is a layout description, not an image scene)
- "Grammar concept" (not visual)

Leave `"assets": []` for selected slides — the generator will populate these.

### Step 3: Dry run

```bash
python3 course/tools/generate_lesson_images.py --repo-root . --lesson {ID} --dry-run
```

Verify that:
- The correct number of images are queued
- Each prompt starts with the watercolour style prefix and includes the scene description
- No prompts contain Thai characters

### Step 4: Generate images

```bash
python3 course/tools/generate_lesson_images.py --repo-root . --lesson {ID}
```

This will:
- Generate PNG images via Gemini 3.1 Flash Image Preview
- Save to `slide-assets/{slide-id}-{slug}.png`
- Update `deck-source.json` with asset entries
- Update `{ID}-asset-provenance.json` with generation metadata

**Expected**: ~10-15 seconds per image. Each image is ~1MB.

If any image fails to generate, the script continues with the remaining images and reports the error. You can re-run with `--regenerate` to retry failed images.

### Step 5: Visually inspect images

Read each generated PNG file to visually verify:
- Watercolour sketch style is consistent
- Scene matches the prompt intent
- No inappropriate content or text in the image
- Images are contextually appropriate for the lesson

If any image is unsatisfactory, update the `imagePrompt` in deck-source.json and re-run:
```bash
python3 course/tools/generate_lesson_images.py --repo-root . --lesson {ID} --regenerate
```

### Step 6: Render PPTX

```bash
python3 course/tools/render_lesson_deck.py --repo-root . --lesson {ID}
```

This will:
- Rebuild the PPTX with images placed bottom-right (below PiP camera zone)
- Preserve `generated-ai` flags and `imagePrompt` values via merge logic
- Run layout validation (fails on text overlaps or PiP intrusions)

Verify that deck-source.json still has the correct number of `generated-ai` entries after rendering:
```bash
grep -c "generated-ai" course/modules/{module}/{lesson}/{ID}-deck-source.json
```

### Step 7: Upload to Google Slides

```bash
python3 course/tools/upload_gslides.py --repo-root . --lesson {ID}
```

This will:
- Delete any previous presentation for this lesson (idempotent)
- Upload the PPTX and convert to native Google Slides
- Verify slide count, page dimensions, Thai text presence
- Share with configured email addresses
- Write `{ID}-gslides-design.json` with the presentation URL

### Step 8: Thai font pass

```bash
python3 course/tools/gslides_font_pass.py --repo-root . --lesson {ID}
```

This swaps Sarabun to Noto Sans Thai Looped on all Thai text runs (U+0E00-U+0E7F). Reports the number of runs restyled.

### Step 9: Report

Report to the user:
1. The Google Slides URL (from the upload output)
2. Number of images generated and which slides they're on
3. Total slide count
4. Font pass result (number of Thai text runs restyled)
5. Any issues or warnings

## Image generation model

- **Model**: `gemini-3.1-flash-image-preview`
- **Auth**: `GEMINI_API_KEY` from `.env` file (Google AI Studio API key)
- **SDK**: `google-genai` Python package
- **Style**: Watercolour sketch, soft brushstrokes, warm earthy palette, hand-drawn travel journal style
- **Cost**: ~$0.045 per image at low resolution

## Image placement on slides

- **Position**: Bottom-right, below PiP camera zone
- **Coordinates**: `left=8.0", top=3.6"`, max `4.8" x 3.4"`
- **Function**: `_place_slide_image()` in `render_lesson_deck.py`
- **Behaviour**: Only places image if `imageUsage == "generated-ai"` and asset file exists. Falls back to text-only gracefully.

## Key files

| File | Purpose |

## Error handling

- If image generation fails for a slide: skip it and continue. Report which slides failed.
- If PPTX rendering fails with layout validation: check if images cause PiP overlap. Adjust image position or remove the offending image.
- If Google Slides upload fails: check OAuth token (may need re-auth at `localhost:8085`). The token is cached at `course/.gslides-token.json`.
- If font pass fails: the presentation still works, just with Sarabun instead of Noto Sans Thai Looped for Thai text. Report and continue.
- Never force success when a step fails. Report the error and let the user decide.
